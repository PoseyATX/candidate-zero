## Product recovery (`695d9b4`) — intended behavior

### Inspect → PLAY (law)

1. Tap any playable face (including locked for inspect).
2. Bottom sheet: title, tagline, full `card.d`, odds, risk, cost.
3. **Play** commits; locked → disabled with reason.
4. Field: Play → ground picker → ground commit.
5. **No** `playFromHand` on face click alone.

### Chrome

- HUD leading: persona (no bare “The”) + issue line.
- Goal strip: labels Goal / Now / Next.
- Setup: How to Play beside Begin primary.
- Card name CSS slightly larger.

### Automation

- `scripts/ui-smoke.mjs`: setup howto, detail open, PLAY, field grounds, **no infinite shop**.
- `scripts/a11y-audit.mjs`: card-detail state; terminal loop uses inspect path.

### Known residual risks

- Owner has **not** confirmed recovery on device.
- `paint()` closes detail on full repaint (by design so End Week isn’t blocked); if something paints while reading, sheet closes — watch for that.
- Playwright a11y/smoke can hang if multiple previews/ports; use distinct `SMOKE_PORT` / `A11Y_PORT`.
- Rival copy may still feel unclear; engine teeth live in field odds (`rivalRap` / calendar path) — verify against `docs/GAME-FLOW.md` and engine, not marketing copy.
